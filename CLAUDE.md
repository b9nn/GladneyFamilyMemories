# Gladney Family Tree — CLAUDE.md
> Last updated: 2026-04-18 | Branch: main

Private family memories web app at **mrtag.com**. Family members store, browse, and connect written stories (vignettes), photos, video, audio, and documents — tied together with a family tree, timeline, and full-text search.

## Critical Conventions

### TypeScript
- **Never use `any`** — use proper types or `unknown` with a comment explaining why
- All files must be `.ts` or `.tsx` — no `.js` or `.jsx`
- Strict mode enabled: `noUnusedLocals`, `noUnusedParameters`
- Define shared types in `src/types/api.ts`; feature-local types in `features/<name>/types.ts`

### Styling
- **Always Tailwind utility classes** — never inline styles, never raw CSS files
- Use `cn()` from `src/lib/utils/utils.ts` for conditional class merging
- Follow shadcn/ui patterns for all UI components
- `src/components/ui/` is auto-generated by shadcn — **do not hand-edit those files**

### Data Fetching
- **All API calls go through `src/lib/api/`** — never call axios directly in components
- **TanStack Query for ALL server state** — no raw `useState + useEffect` for server data
- Mutations must invalidate relevant query keys on success

### State Management
- Server state → TanStack Query
- Client state → Zustand stores
- **No new React Context** for state

### Backend
- New models → `models.py`, new schemas → `schemas.py`, new endpoints → `main.py`
- **Every schema change gets an Alembic migration** — never run `create_all()` in production
- Follow dependency injection pattern: `get_db`, `get_current_user` as FastAPI dependencies
- All endpoints must have an explicit Pydantic `response_model`

### Components
- Page components under 200 lines — extract into `features/<name>/components/`
- Each feature: `src/features/<name>/` with `components/`, `hooks/`, `page.tsx`

### Git
- Commit messages: imperative mood ("Add vignette card component")
- One logical change per commit
- Push to `main` triggers Fly.io deploy via GitHub Actions (`fly-deploy.yml`)

## Key Architecture Decisions

**Same-origin API in production**: Frontend and backend run on the same Fly.io app (`gladney-family-tree`, region `iad`). `VITE_API_URL=""` in production so API calls go to `/api/...` on the same origin. Never hardcode an absolute API URL in components. CD is via GitHub Actions (`fly-deploy.yml`) — push to `main` triggers a deploy automatically.

**Dual DB mode**: `database.py` detects `DATABASE_URL` env var — if set → PostgreSQL (prod); if absent → SQLite file at `./tag_diary.db` (dev). Use SQLAlchemy abstractions only, no raw SQL.

**Storage abstraction**: `storage.py` checks `USE_CLOUD_STORAGE=true` → Cloudflare R2; else → local `/uploads/` directory. Never write files directly to disk in production code.

**HEIC handling**: iPhone photos are uploaded as HEIC. `storage.py` must convert HEIC → JPEG using `pillow-heif` before storing. Always convert before generating thumbnails.

**R2 presigned URLs**: File URLs from R2 are presigned and expire in 1 hour. Never cache them in localStorage or long-lived state. Always re-fetch when needed.

**Admin scope**: Admin-only routes are prefixed `/api/admin/` and require `is_admin: true` in the JWT payload. Check `get_current_admin_user` dependency — don't use `get_current_user` for admin routes.

**TipTap for rich text**: Vignettes use TipTap v2 for full rich text (bold, italic, headings, lists, links). TipTap output is stored as JSON in the `content` column. Always render with TipTap's `generateHTML()`, never dangerouslySetInnerHTML with raw strings.

**Invite-code registration**: Registration is gated by invite codes. Invite codes are single-use and created by admins. The registration endpoint validates the code before creating the user.

**Predefined tags only**: Tag categories are admin-controlled. Users cannot create free-form tags — they pick from the admin-defined list. Tag categories: `person`, `place`, `event`, `topic`.

## Sensitive Areas / Gotchas

- `family_members` uses a junction table `family_relationships` for parent/child, spouse, sibling — **be careful with cascade deletes** (deleting a member should not cascade-delete the other member in a relationship)
- R2 presigned URLs expire in 1 hour — **do not cache them**
- The `source` column on `files` distinguishes file uploads (`"files"`) from vignette attachments (`"vignettes"`) — always set this when inserting
- `album_photos` is a many-to-many — a photo can belong to multiple albums AND exist outside albums
- Video uploads require size validation — set a max file size in storage.py (e.g., 500MB)
- In local dev, `USE_CLOUD_STORAGE` is not set, so all uploads go to `backend/uploads/` — never commit this directory
- `content_tags` is polymorphic: `content_type` = `"vignette"` | `"photo"` | `"audio"` | `"file"` — always filter by both `content_type` and `content_id`
