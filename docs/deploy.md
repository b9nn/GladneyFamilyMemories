# Deployment Guide

## Architecture

Production runs as a **single Render web service**: FastAPI serves both the API (`/api/*`) and the built React frontend (SPA fallback via `frontend/dist/`). No separate hosting for the frontend.

```
mrtag.com → Render web service (FastAPI + uvicorn)
                ├── /api/* → FastAPI routes
                ├── /uploads/* → local disk (Render persistent disk)
                └── /* → React SPA (frontend/dist/index.html)
```

## First-time setup

### 1. Cloudflare R2 bucket

1. Log in to [dash.cloudflare.com](https://dash.cloudflare.com) → R2 → Create bucket: `gladneyfamilymemories`
2. Settings → Public Access → Enable (or use presigned URLs only — already the default)
3. Manage API Tokens → Create token with `Object Read & Write` on that bucket
4. Note down: **Endpoint URL**, **Access Key ID**, **Secret Access Key**

### 2. Render — create service from `render.yaml`

1. Push this repo to GitHub
2. [dashboard.render.com](https://dashboard.render.com) → New → Blueprint → connect repo → select `render.yaml`
3. Render will provision:
   - PostgreSQL database (`gladney-family-db`)
   - Web service (`gladney-family-tree`)
4. After provisioning, go to web service → **Environment** and fill in the `sync: false` vars:
   - `S3_ENDPOINT_URL` — from Cloudflare R2 (e.g. `https://<account-id>.r2.cloudflarestorage.com`)
   - `S3_ACCESS_KEY_ID` — R2 API token access key
   - `S3_SECRET_ACCESS_KEY` — R2 API token secret
   - `ADMIN_EMAIL` — your email (for admin notifications)
   - `SMTP_HOST`, `SMTP_USER`, `SMTP_PASSWORD` — optional (email notifications)

### 3. Custom domain

1. Render → your service → Settings → Custom Domain → Add `mrtag.com`
2. Follow the CNAME/ALIAS instructions in your DNS provider

### 4. Bootstrap first admin

After the first deploy, create the admin account:

```bash
curl -X POST https://mrtag.com/api/admin/bootstrap \
  -H "Content-Type: application/json" \
  -d '{
    "secret": "<your BOOTSTRAP_SECRET from Render env>",
    "username": "tom",
    "password": "<your password>",
    "full_name": "Tom Gladney"
  }'
```

Copy the `access_token` from the response — paste it into your browser's localStorage:

```js
localStorage.setItem('token', '<access_token>');
```

Then refresh the page. You're logged in as admin.

### 5. Generate first invite code (for other family members)

In the Admin panel → Invite Codes → Generate code → copy and share with each family member.

## GitHub Actions

### CI (runs on every push)

`.github/workflows/ci.yml` runs:
- Frontend: `npm run typecheck` → `npm run lint` → `npm run build`
- Backend: `pip install` → import check → `pytest`

### Auto-deploy (runs on push to `main`)

1. Render → your service → Settings → Deploy Hook → copy the URL
2. GitHub repo → Settings → Secrets → `RENDER_DEPLOY_HOOK_URL` → paste URL
3. Every push to `main` triggers a Render deploy automatically

## Manual deploy

```bash
# Merge clean-start → main
git checkout main
git merge clean-start
git push origin main
# GitHub Actions triggers the Render deploy hook automatically
```

## Environment variables reference

| Variable | Required | Description |
|---|---|---|
| `SECRET_KEY` | ✅ | JWT signing secret (auto-generated by Render) |
| `BOOTSTRAP_SECRET` | ✅ | One-time secret for creating first admin |
| `DATABASE_URL` | ✅ | PostgreSQL connection string (set by Render) |
| `USE_CLOUD_STORAGE` | ✅ | Set to `true` in production |
| `S3_ENDPOINT_URL` | ✅ | Cloudflare R2 endpoint |
| `S3_ACCESS_KEY_ID` | ✅ | R2 API access key |
| `S3_SECRET_ACCESS_KEY` | ✅ | R2 API secret |
| `S3_BUCKET_NAME` | ✅ | R2 bucket name |
| `SITE_URL` | ✅ | `https://mrtag.com` |
| `ADMIN_EMAIL` | — | Admin notification recipient |
| `SMTP_HOST` | — | SMTP server for email notifications |
| `SMTP_PORT` | — | Default: `587` |
| `SMTP_USER` | — | SMTP username |
| `SMTP_PASSWORD` | — | SMTP password |
| `FROM_EMAIL` | — | Sender address for notifications |
